# Development Standards

## Цель

Единые стандарты разработки проекта.

## Code Style

- Следование современным Best Practices.
- ESLint.
- Prettier.
- Осмысленные имена переменных, функций и компонентов.
- Читаемость кода важнее краткости.
- Единый стиль оформления проекта.

## CSS

- SCSS.
- Методология BEM.
- Компонентный подход.
- Минимизация дублирования стилей.

## Общие правила

1. Минимум зависимостей.
2. Использовать только необходимые библиотеки.
3. Типовые задачи решать проверенными пакетами.
4. Уникальный интерфейс разрабатывать самостоятельно.
5. Перед добавлением новой библиотеки оценивать её необходимость.
6. Соблюдать единые соглашения по именованию и структуре проекта.
